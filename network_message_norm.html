Network Message Norm Documentation

Overview

This document outlines the standard message format utilised by the server in our network application. The message norm is designed to ensure consistency, efficiency, and ease of parsing across all network communications.

Message Format

Each message follows a specific structure to maintain uniformity. The general format is: 

element:value-element:value-ip:ip_address

Components

  • Element: Represents a key in the message. It is a string that describes the type of data.
  • Value: The value associated with the corresponding element. It can be a string, number, or any other data type as per requirement.
  • IP Address: A special element representing the IP address of the sender or target, depending on the context.

Separator

  • Elements and their values are separated by a colon :.
  • Different key-value pairs are separated by a hyphen -.

Rules

  1. Order: The order of key-value pairs in a message is not fixed and can vary.
  2. Uniqueness: Each key within a single message must be unique. If a key is repeated, the last occurrence is considered valid.
  3. IP Address Format: The IP address should follow the standard IPv4 or IPv6 formatting.
  4. Special Characters: Avoid using the colon : and hyphen - within keys and values, as these are reserved for separating message components.

Example Messages

  1. Basic Message: 
    x:0-y:10-hp:40-ip:127.0.0.1:13151
    This message contains three data elements (x, y, hp) and the IP address.
  2. Variable Elements: 
    status:active-id:12345-ip:192.168.1.10:13152
    A different set of data elements showing the flexibility of the message format.

Parsing

Messages should be parsed according to the defined separators. Ensure that the parser is robust enough to handle variations in the order of elements and the inclusion of different data types as values.
