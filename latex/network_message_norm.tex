\hypertarget{network_message_norm_overview}{}\doxysection{Overview}\label{network_message_norm_overview}
This document outlines the standard message format utilised by the server in our network application. The message norm is designed to ensure consistency, efficiency, and ease of parsing across all network communications.\hypertarget{network_protocol_rfc_message_format}{}\doxysection{Message Format}\label{network_protocol_rfc_message_format}
Each message follows a specific structure to maintain uniformity. The general format is\+: \begin{DoxyVerb}element:value-element:value-ip:ip_address
\end{DoxyVerb}
\hypertarget{network_protocol_rfc_components}{}\doxysubsection{Components}\label{network_protocol_rfc_components}

\begin{DoxyItemize}
\item {\bfseries{Element}}\+: Represents a key in the message. It is a string that describes the type of data.
\item {\bfseries{Value}}\+: The value associated with the corresponding element. It can be a string, number, or any other data type as per requirement.
\item {\bfseries{IP Address}}\+: A special element representing the IP address of the sender or target, depending on the context.
\end{DoxyItemize}\hypertarget{network_message_norm_separator}{}\doxysubsection{Separator}\label{network_message_norm_separator}

\begin{DoxyItemize}
\item Elements and their values are separated by a colon {\bfseries{\+:}}.
\item Different key-\/value pairs are separated by a hyphen {\bfseries{-\/}}.
\end{DoxyItemize}\hypertarget{network_message_norm_rules}{}\doxysection{Rules}\label{network_message_norm_rules}

\begin{DoxyEnumerate}
\item {\bfseries{Order}}\+: The order of key-\/value pairs in a message is not fixed and can vary.
\item {\bfseries{Uniqueness}}\+: Each key within a single message must be unique. If a key is repeated, the last occurrence is considered valid.
\item {\bfseries{IP Address Format}}\+: The IP address should follow the standard IPv4 or IPv6 formatting.
\item {\bfseries{Special Characters}}\+: Avoid using the colon {\bfseries{\+:}} and hyphen {\bfseries{-\/}} within keys and values, as these are reserved for separating message components.
\end{DoxyEnumerate}\hypertarget{network_message_norm_example_messages}{}\doxysection{Example Messages}\label{network_message_norm_example_messages}

 This message contains three data elements ({\bfseries{x}}, {\bfseries{y}}, {\bfseries{hp}}) and the IP address.
\item {\bfseries{Variable Elements}}\+: \begin{DoxyVerb}status:active-id:12345-ip:192.168.1.10:13152 \end{DoxyVerb}
 A different set of data elements showing the flexibility of the message format.
\end{DoxyEnumerate}\hypertarget{network_message_norm_parsing}{}\doxysection{Parsing}\label{network_message_norm_parsing}
